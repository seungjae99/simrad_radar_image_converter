ROS Noetic C++17 OpenGL 3.3 Qt5

radar_com_img_range

A ROS node that receives raw polar sweep data from the Simrad Halo20+ marine radar, reconstructs a full 360° Cartesian image using offscreen GPU rendering, and publishes it as a compressed image for downstream robotics pipelines.

Overview

Marine radar sensors like the Simrad Halo20+ stream data as a sequence of narrow angular slices called sectors. Each sector contains a 1-D array of echo intensities measured along a radial sweep. To use this data with standard image-based algorithms — such as radar odometry or SLAM — the polar representation must be converted to a rectilinear Cartesian image.

radar_com_img_range performs this conversion entirely on the GPU using a Qt5 offscreen OpenGL context and a custom GLSL fragment shader. After accumulating enough sectors to complete one full revolution, it renders the result into a framebuffer object, reads the pixels back, and publishes the grayscale image as a ROS sensor_msgs/CompressedImage (PNG or JPEG).

One image is published per full 360° revolution. The wrap-around is detected by monitoring the sign-change of the angle_start delta between consecutive sectors (Δ > π radians triggers a flush).

Core Features
  • 1
    Dynamic framebuffer sizing. The OpenGL FBO is created on the first received message, sized to echo_count × 2 pixels (diameter = full radial resolution), clamped to [64, 2048]. No configuration required.
  • 2
    GPU polar-to-Cartesian projection. The GLSL fragment shader converts each output pixel's Cartesian coordinates to polar (r, theta), samples the correct sector texture via UV mapping, and discards pixels outside the angular range of that sector. All sectors are composited in a single render pass.
  • 3
    Wrap-around boundary correction. Sectors that straddle the 0/2π boundary are handled in the shader via a separate branch, ensuring the rendered circle has no seam artifact at north (top of image).
  • 4
    Selectable compressed output. The output image is encoded as PNG (lossless, default) or JPEG (configurable quality, 0–100). This reduces ROS topic bandwidth compared to raw sensor_msgs/Image.
  • 5
    Radar lifecycle management. On startup the node commands the sensor to transmit; on shutdown (ROS OK check in Qt timer loop) it commands standby. This prevents the radar from running unattended if the node crashes.
  • 6
    Headless operation. Qt5 is forced into offscreen mode via QT_QPA_PLATFORM=offscreen, making the node safe to run on robots without a display server. The Qt event loop drives ros::spinOnce() through a 10 ms QTimer to keep ROS callbacks alive.

Design Notes
Why Qt5 OpenGL instead of OpenCV warpPolar?
GPU-side angular clipping via the fragment shader avoids seam gaps between sectors and handles arbitrary angular increments without resampling artifacts. Each sector is projected exactly once.
Grayscale output
Echo intensity is a scalar value. Encoding as grayscale halves the memory footprint and is the expected input format for radar-based odometry and feature-matching algorithms.
FBO size is echo-driven
Setting FBO size to echo_count × 2 ensures one output pixel per range bin along any radial line, preserving the sensor's native range resolution without upsampling.
Sector angle edge interpretation
angle_start is treated as the leading edge of the sweep block, with width angle_increment × H. A small epsilon (1e-4 rad) is added to both boundaries to prevent hairline gaps at sector edges.
Relationship to Other Nodes

This node is the most production-ready variant in the package. It supersedes radar_img_range (which publishes raw sensor_msgs/Image) by adding compressed transport support, and supersedes radar_img_crop (which requires a fixed sensor_range_max parameter) by deriving the FBO size automatically from the sensor data.

The published CompressedImage is intended as direct input to radar odometry or scan-matching algorithms running elsewhere in the ROS graph. The fixed frame_id: map default assumes a stationary or map-referenced sensor mount; override via the ~frame_id parameter when used on a moving platform.
