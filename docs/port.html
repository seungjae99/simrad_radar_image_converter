01 개발 배경 및 문제 정의
프로젝트 컨텍스트

선박해양플랜트연구소(KRISO) 의뢰 과제로, GPS를 신뢰할 수 없는 극한 해상 환경에서 Simrad Halo20+ 해상 레이더 단독으로 항법을 수행하는 오도메트리 추정 시스템을 개발하는 프로젝트였습니다. 본 드라이버 확장 노드는 해당 파이프라인의 최초 데이터 진입점 역할을 담당합니다.

문제 1 — 극좌표 전용 출력: 기존 드라이버(marine_sensor_msgs/RadarSector)는 극좌표(r, θ) 형식의 강도값만 제공했습니다. 오도메트리 알고리즘과 Rviz 기반 실시간 모니터링에 직접 활용하려면 직교좌표계 이미지로의 변환이 필수였습니다.
문제 2 — 사후(Post) 변환의 한계: 기록 후 오프라인 변환 방식은 실해역 실험 중 즉각적인 데이터 확인과 이상 감지가 불가능합니다. 파이프라인 전단에서 실시간으로 처리된 데이터가 필요했습니다.
해결 방향: 드라이버 단에 ROS 노드를 추가하여 극좌표 스캔을 수신하는 즉시 Qt5 Offscreen OpenGL FBO로 렌더링, 직교좌표 이미지를 생성·퍼블리시하는 확장 레이어를 설계했습니다.

03 핵심 구현 상세
Qt5 Offscreen OpenGL 렌더링 구조

헤드리스(Headless) 환경에서 GPU 가속 렌더링을 수행하기 위해 Qt5의 QOffscreenSurface와 QOpenGLFramebufferObject(FBO)를 활용했습니다. 디스플레이 없이 동작해야 하는 로봇 온보드 컴퓨터 환경을 위해 QT_QPA_PLATFORM=offscreen 및 QT_OPENGL=software 환경 변수를 설정합니다.

1
섹터 누적 (addSector): RadarSector 콜백마다 각도-강도 쌍을 QImage (Grayscale8)에 극좌표로 누적합니다.
2
360도 완성 감지 (Wraparound): 섹터 각도가 이전 값보다 작아지는 시점(wraparound)을 감지하여 한 회전 완료를 판단합니다.
3
GLSL 셰이더 렌더링 (renderFullCircleGray): 누적된 극좌표 텍스처를 FBO에 바인딩하고, GLSL 프래그먼트 셰이더로 직교좌표 → 극좌표 역변환 샘플링을 수행하여 Cartesian 이미지를 생성합니다.
4
ROS 메시지 변환 및 퍼블리시: FBO 픽셀 데이터를 cv_bridge로 읽어 sensor_msgs/Image 또는 CompressedImage로 변환 후 퍼블리시합니다.
GLSL 프래그먼트 셰이더 — 극좌표 역변환 샘플링

Cartesian 이미지의 각 픽셀 (x, y)를 중심 기준으로 정규화한 뒤, atan2와 length로 극좌표 (θ, r)를 역산하여 원본 극좌표 텍스처를 샘플링합니다. 

// Fragment Shader — Polar-to-Cartesian inverse mapping
uniform sampler2D uPolarTex;
in      vec2      vTexCoord;
out     vec4      fragColor;

void main() {
  // NDC → 중심 기준 좌표 (-1 ~ +1)
  vec2 p = vTexCoord * 2.0 - 1.0;

  float r     = length(p);            // 반경 (0 ~ 1)
  float theta = atan(p.x, p.y);       // 방위각 (-π ~ +π)

  if (r > 1.0) { fragColor = vec4(0.0); return; }

  // 극좌표 UV 산출 (각도 정규화 → 0 ~ 1)
  vec2 polarUV = vec2(
    r,
    (theta + 3.14159265) / (2.0 * 3.14159265)
  );

  float intensity = texture(uPolarTex, polarUV).r;
  fragColor = vec4(vec3(intensity), 1.0);
}
FBO 크기 결정 전략

radar_img_crop은 파라미터 sensor_range_max로 FBO 크기를 노드 시작 시 사전 결정합니다. radar_img_range / com은 첫 번째 RadarSector 메시지의 실제 에코 카운트에서 FBO 크기를 동적으로 산출하여 센서 설정 변경에 유연하게 대응합니다. FBO 크기는 [64, 4096]으로 클램프됩니다.

설계 의도: 범위 파라미터를 실시간으로 변경해도 노드를 재시작하지 않고 자동으로 해상도가 갱신됩니다.
레이더 생명주기 관리

노드 시작 시 레이더를 transmit 상태로, 종료 시 standby로 자동 전환합니다. Python 스크립트(radar_set_state.py)로 수동 제어도 가능합니다.

제약: QT_QPA_PLATFORM=offscreen 및 QT_OPENGL=software 환경 변수는 헤드리스 동작의 필수 조건으로 절대 제거하면 안 됩니다.
압축 이미지 노드 — 대역폭 최적화

radar_com_img_range는 동일한 렌더링 파이프라인에 JPEG/PNG 압축 레이어를 추가합니다. 실해역 실험에서 유선 LAN이 아닌 무선 통신 환경의 대역폭 제약을 고려한 설계입니다. 

// cv_bridge → CompressedImage 변환 핵심 흐름
std::vector<uchar> buf;
std::vector<int>   params = { cv::IMWRITE_JPEG_QUALITY, jpeg_quality_ };

cv::imencode(".jpg", cv_img, buf, params);

sensor_msgs::CompressedImage msg;
msg.header  = header;
msg.format  = "jpeg";
msg.data    = buf;
pub_.publish(msg);
파라미터 _compression_format(jpeg / png)와 _jpeg_quality(0~100)로 압축 방식을 런타임에 조정할 수 있습니다.
